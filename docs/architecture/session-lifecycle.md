# Жизненный цикл сессии

Сессия является основной единицей аналитического контекста. Она связывает несколько снов во времени и позволяет системе видеть повторения, напряжения и изменения.

## Состояния

- `active` - текущая рабочая сессия пользователя.
- `paused` - сессия временно не развивается, но может быть продолжена.
- `closed` - завершенная сессия, доступная для истории и summary.

## Основные правила

- У пользователя не должно быть больше одной активной сессии.
- Новый сон должен привязываться к текущей активной сессии или создавать новую сессию через backend service.
- Сессия закрывается автоматически после 72 часов неактивности.
- Закрытие сессии не должно изменять исходные сны.
- Summary можно пересобрать детерминированно.

## Владение логикой

Session lifecycle принадлежит backend services. Telegram bot, Mini App и AI layer не должны вычислять, создавать или закрывать сессии напрямую.

```text
user message
    ↓
backend intake service
    ↓
resolve active session
    ↓
persist dream
    ↓
update deterministic summary
```

## Причина

Сновидения редко имеют изолированный смысл. Повторы образов, эмоциональная тональность и смена мотивов становятся видимыми только в последовательности. Поэтому система должна проектироваться вокруг сессий, а не вокруг отдельных сообщений.
